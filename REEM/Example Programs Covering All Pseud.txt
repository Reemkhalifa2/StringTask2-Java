Program 6 — Voting System
Covers: CREATE SET, ADD TO SET, SET CONTAINS, CREATE MAP, SET MAP KEY TO, GET FROM MAP, MAP CONTAINS KEY
START
BEGIN
   CREATE SET VOTED_IDS
   CREATE MAP VOTE_COUNT

   SET MAP VOTE_COUNT "Candidate A" TO 0
   SET MAP VOTE_COUNT "Candidate B" TO 0
   SET MAP VOTE_COUNT "Candidate C" TO 0

   REPEAT FROM I = 1 TO 5
      DISPLAY "Enter Voter ID"
      READ VOTER_ID

      IF SET CONTAINS VOTED_IDS VOTER_ID THEN
         DISPLAY "Already voted"
      ELSE
         ADD VOTER_ID TO SET VOTED_IDS

         DISPLAY "Enter your choice (A, B or C)"
         READ CHOICE

         IF CHOICE EQUALS "A" THEN
            SET MAP VOTE_COUNT "Candidate A" TO GET FROM MAP VOTE_COUNT "Candidate A" + 1
         ELSE IF CHOICE EQUALS "B" THEN
            SET MAP VOTE_COUNT "Candidate B" TO GET FROM MAP VOTE_COUNT "Candidate B" + 1
         ELSE IF CHOICE EQUALS "C" THEN
            SET MAP VOTE_COUNT "Candidate C" TO GET FROM MAP VOTE_COUNT "Candidate C" + 1
         ELSE
            DISPLAY "Invalid choice"
         END IF

         DISPLAY "Vote recorded"
      END IF
   END REPEAT

   DISPLAY "--- Final Results ---"
   SET KEYS TO GET ALL KEYS VOTE_COUNT
   REPEAT FROM I = 1 TO SIZE OF KEYS
      SET CANDIDATE TO KEYS[I]
      DISPLAY CANDIDATE + ": " + GET FROM MAP VOTE_COUNT CANDIDATE + " votes"
   END REPEAT
END
STOP

// Output: --- Final Results ---
//         Candidate A: 2 votes
//         Candidate B: 2 votes
//         Candidate C: 1 votes

## 1. PROGRAM FLOW

**START / STOP / BEGIN / END**
> Every program must open and close properly. START and STOP wrap the whole program. BEGIN and END wrap the steps inside.

```
START
BEGIN
   DISPLAY "Welcome to my program"
END
STOP

// Output: Welcome to my program
```

---

## 2. DATA & VARIABLES

**SET — Assign a value**
> SET is how you store something in memory so you can use it later.

```
START
BEGIN
   SET NAME TO "Sara"
   SET AGE TO 20
   SET IS_STUDENT TO TRUE

   DISPLAY NAME + " is " + AGE + " years old"
END
STOP

// Output: Sara is 20 years old
```

---

**NUMBER**
> Stores a numeric value — used for calculations.

```
START
BEGIN
   SET PRICE TO 100
   SET DISCOUNT TO 20
   SET FINAL TO PRICE - DISCOUNT

   DISPLAY "Final price: " + FINAL
END
STOP

// Output: Final price: 80
```

---

**TEXT**
> Stores words or sentences.

```
START
BEGIN
   SET GREETING TO "Hello"
   SET CITY TO "Muscat"

   DISPLAY GREETING + " from " + CITY
END
STOP

// Output: Hello from Muscat
```

---

**BOOLEAN**
> Stores only TRUE or FALSE — used for yes/no decisions.

```
START
BEGIN
   SET IS_RAINING TO TRUE

   IF IS_RAINING EQUALS TRUE THEN
      DISPLAY "Take an umbrella"
   ELSE
      DISPLAY "No umbrella needed"
   END IF
END
STOP

// Output: Take an umbrella
```

---

## 3. INPUT & OUTPUT

**DISPLAY — Show output**
> DISPLAY prints something to the screen for the user to see.

```
START
BEGIN
   DISPLAY "Good morning"
   DISPLAY "Today is a great day"
END
STOP

// Output: Good morning
//         Today is a great day
```

---

**READ — Take input**
> READ pauses the program and waits for the user to type something.

```
START
BEGIN
   DISPLAY "What is your name?"
   READ NAME
   DISPLAY "Nice to meet you " + NAME
END
STOP

// Input:  Ali
// Output: Nice to meet you Ali
```

---

## 4. CONDITIONS

**IF / THEN / ELSE / END IF**
> IF checks whether something is true. If it is, it runs the THEN block. Otherwise it runs the ELSE block.

```
START
BEGIN
   DISPLAY "Enter your score"
   READ SCORE

   IF SCORE GREATER THAN OR EQUAL TO 50 THEN
      DISPLAY "You passed"
   ELSE
      DISPLAY "You failed"
   END IF
END
STOP

// Input:  75
// Output: You passed

// Input:  30
// Output: You failed
```

---

## 5. COMPARISON WORDS

**EQUALS / NOT EQUALS / GREATER THAN / LESS THAN / GREATER THAN OR EQUAL TO / LESS THAN OR EQUAL TO**
> These are used inside IF conditions to compare two values.

```
START
BEGIN
   SET TEMPERATURE TO 38

   IF TEMPERATURE GREATER THAN 37 THEN
      DISPLAY "You have a fever"
   ELSE IF TEMPERATURE EQUALS 37 THEN
      DISPLAY "Normal temperature"
   ELSE
      DISPLAY "Below normal"
   END IF
END
STOP

// Output: You have a fever
```

---

## 6. LOGICAL CONNECTORS

**AND — Both must be true**
> Use AND when two conditions must both be satisfied.

```
START
BEGIN
   SET AGE TO 20
   SET HAS_ID TO TRUE

   IF AGE GREATER THAN OR EQUAL TO 18 AND HAS_ID EQUALS TRUE THEN
      DISPLAY "Entry allowed"
   ELSE
      DISPLAY "Entry denied"
   END IF
END
STOP

// Output: Entry allowed
```

---

**OR — At least one must be true**
> Use OR when either one condition or the other being true is enough.

```
START
BEGIN
   SET DAY TO "Saturday"

   IF DAY EQUALS "Saturday" OR DAY EQUALS "Sunday" THEN
      DISPLAY "It is the weekend"
   ELSE
      DISPLAY "It is a weekday"
   END IF
END
STOP

// Output: It is the weekend
```

---

**NOT — Reverse the condition**
> NOT flips TRUE to FALSE and FALSE to TRUE.

```
START
BEGIN
   SET IS_LOGGED_IN TO FALSE

   IF NOT IS_LOGGED_IN THEN
      DISPLAY "Please log in first"
   ELSE
      DISPLAY "Welcome back"
   END IF
END
STOP

// Output: Please log in first
```

---

## 7. LOOPS

**REPEAT FROM / TO / END REPEAT — Fixed repetition**
> Use this when you know exactly how many times to repeat.

```
START
BEGIN
   DISPLAY "Countdown:"

   REPEAT FROM I = 1 TO 5
      DISPLAY I
   END REPEAT

   DISPLAY "Done"
END
STOP

// Output: 1
//         2
//         3
//         4
//         5
//         Done
```

---

**REPEAT WHILE / END REPEAT — Conditional repetition**
> Use this when you want to keep repeating until something changes.

```
START
BEGIN
   SET COUNT TO 3

   REPEAT WHILE COUNT GREATER THAN 0
      DISPLAY "Attempts left: " + COUNT
      SET COUNT TO COUNT - 1
   END REPEAT

   DISPLAY "No more attempts"
END
STOP

// Output: Attempts left: 3
//         Attempts left: 2
//         Attempts left: 1
//         No more attempts
```

**LIST — Collection of values**
> A list holds multiple values under one name.

```
START
BEGIN
   SET FRUITS TO LIST ["Apple", "Banana", "Orange"]
   DISPLAY "First fruit: " + FRUITS[1]
   DISPLAY "Total fruits: " + SIZE OF FRUITS
END
STOP

// Output: First fruit: Apple
//         Total fruits: 3
```

**ADD — Insert a value**
> ADD puts a new item into a list.

```

BEGIN

   DISPLAY SIZE OF NAMES
END

// Output: 3
```

> REMOVE takes an item out of a list.

END

// Output: 2
```

```

## 9. SETS

**CREATE SET / ADD TO SET / REMOVE FROM SET / SET CONTAINS / SET SIZE / CLEAR SET**
> A set is like a list but it never stores the same value twice.

```
START
BEGIN
   CREATE SET ATTENDEES

   ADD "Ali" TO SET ATTENDEES
   ADD "Sara" TO SET ATTENDEES
   ADD "Ali" TO SET ATTENDEES   // duplicate — ignored

   DISPLAY "Total attendees: " + SET SIZE OF ATTENDEES

   IF SET CONTAINS ATTENDEES "Sara" THEN
      DISPLAY "Sara is attending"
   END IF

   REMOVE FROM SET ATTENDEES "Sara"
   DISPLAY "After removal: " + SET SIZE OF ATTENDEES

   CLEAR SET ATTENDEES
   DISPLAY "After clearing: " + SET SIZE OF ATTENDEES
END
STOP

// Output: Total attendees: 2
//         Sara is attending
//         After removal: 1
//         After clearing: 0
```

---

**MERGE SETS — Union**
> Combines two sets into one, keeping only unique values.

```
START
BEGIN
   CREATE SET GROUP_A
   ADD "Ali" TO SET GROUP_A
   ADD "Sara" TO SET GROUP_A

   CREATE SET GROUP_B
   ADD "Sara" TO SET GROUP_B
   ADD "Tom" TO SET GROUP_B

   CREATE SET ALL_MEMBERS
   SET ALL_MEMBERS TO MERGE SETS GROUP_A AND GROUP_B

   DISPLAY "Total unique members: " + SET SIZE OF ALL_MEMBERS
   // Sara appears in both but is only counted once
END
STOP

// Output: Total unique members: 3
```

---

**INTERSECT SETS — Common values only**
> Keeps only values that appear in BOTH sets.

```
START
BEGIN
   CREATE SET TEAM_A
   ADD "Python" TO SET TEAM_A
   ADD "Java" TO SET TEAM_A
   ADD "SQL" TO SET TEAM_A

   CREATE SET TEAM_B
   ADD "Java" TO SET TEAM_B
   ADD "SQL" TO SET TEAM_B
   ADD "Excel" TO SET TEAM_B

   CREATE SET SHARED
   SET SHARED TO INTERSECT SETS TEAM_A AND TEAM_B

   DISPLAY "Shared skills: " + SHARED
END
STOP

// Output: Shared skills: Java, SQL
```

---

**DIFFERENCE OF SETS — Unique to one set**
> Keeps only values from the first set that do NOT appear in the second.

```
START
BEGIN
   CREATE SET ALL_STUDENTS
   ADD "Ali" TO SET ALL_STUDENTS
   ADD "Sara" TO SET ALL_STUDENTS
   ADD "Tom" TO SET ALL_STUDENTS

   CREATE SET SUBMITTED
   ADD "Ali" TO SET SUBMITTED
   ADD "Tom" TO SET SUBMITTED

   CREATE SET NOT_SUBMITTED
   SET NOT_SUBMITTED TO DIFFERENCE OF SETS ALL_STUDENTS AND SUBMITTED

   DISPLAY "Did not submit: " + NOT_SUBMITTED
END
STOP

// Output: Did not submit: Sara
```

## 10. HASHMAPS

**CREATE MAP / SET MAP KEY TO / GET FROM MAP / MAP CONTAINS KEY / REMOVE FROM MAP**
> A map stores information as pairs — a key and its value. Like a dictionary.

```
START
BEGIN
   CREATE MAP CONTACTS

   SET MAP CONTACTS "Ali" TO "0501234567"
   SET MAP CONTACTS "Sara" TO "0559876543"

   READ NAME

   IF MAP CONTAINS KEY CONTACTS NAME THEN
      DISPLAY "Number: " + GET FROM MAP CONTACTS NAME
   ELSE
      DISPLAY "Contact not found"
   END IF

   REMOVE FROM MAP CONTACTS "Ali"
   DISPLAY "Ali removed"
END
STOP

// Input:  Sara
// Output: Number: 0559876543
//         Ali removed
```

      DISPLAY "Enter your PIN"
      READ PIN

      IF PIN EQUALS CORRECT_PIN THEN
         SET LOGGED_IN TO TRUE
         DISPLAY "Login successful"
      ELSE
         SET ATTEMPTS TO ATTEMPTS - 1
         DISPLAY "Wrong PIN. Attempts left: " + ATTEMPTS
      END IF
   END REPEAT

   IF LOGGED_IN EQUALS TRUE THEN
      DISPLAY "Your balance: " + BALANCE

      DISPLAY "Enter amount to withdraw"
      READ AMOUNT

      IF AMOUNT GREATER THAN BALANCE THEN
         DISPLAY "Insufficient funds"
      ELSE IF AMOUNT LESS THAN OR EQUAL TO 0 THEN
         DISPLAY "Invalid amount"
      ELSE
         SET BALANCE TO BALANCE - AMOUNT
         DISPLAY "Withdrawn: " + AMOUNT
         DISPLAY "Remaining balance: " + BALANCE
      END IF
   ELSE
      DISPLAY "Card blocked. Too many wrong attempts"
   END IF
END
STOP

Problem 8 — Library Book Tracker

A library tracks which books are available and who has borrowed them.START
BEGIN
   CREATE MAP BORROWED
   SET AVAILABLE TO LIST ["The Alchemist", "1984", "Dune", "Brave New World"]

   DISPLAY "--- Library System ---"
   DISPLAY "Available books: " + SIZE OF AVAILABLE

   DISPLAY "Enter your name"
   READ BORROWER

   DISPLAY "Enter book to borrow"
   READ BOOK

   IF MAP CONTAINS KEY BORROWED BOOK THEN
      DISPLAY "Sorry, " + BOOK + " is already borrowed by " + GET FROM MAP BORROWED BOOK
   ELSE
      SET MAP BORROWED BOOK TO BORROWER
      REMOVE BOOK FROM AVAILABLE
      DISPLAY BORROWER + " has borrowed " + BOOK
   END IF

   DISPLAY "Books still available: " + SIZE OF AVAILABLE
   DISPLAY "Books currently borrowed: " + MAP SIZE OF BORROWED

   DISPLAY "Enter book to return"
   READ RETURN_BOOK

   IF MAP CONTAINS KEY BORROWED RETURN_BOOK THEN
      REMOVE FROM MAP BORROWED RETURN_BOOK
      ADD RETURN_BOOK TO AVAILABLE
      DISPLAY RETURN_BOOK + " has been returned"
   ELSE
      DISPLAY "This book was not borrowed"
   END IF

   DISPLAY "Books now available: " + SIZE OF AVAILABLE
END
STOP

Users vote for their favourite option. No one can vote twice. Final results are displayed at the end.START

   SET MAP TALLY "Option B" TO 0
   SET MAP TALLY "Option C" TO 0

   SET VOTE_NUMBER TO 1

      DISPLAY "Enter Voter ID"
      READ VOTER_ID

         DISPLAY "You have already voted"

         IF CHOICE EQUALS "A" THEN
            SET MAP TALLY "Option A" TO GET FROM MAP TALLY "Option A" + 1
            SET VOTE_NUMBER TO VOTE_NUMBER + 1
         ELSE IF CHOICE EQUALS "B" THEN
            SET MAP TALLY "Option B" TO GET FROM MAP TALLY "Option B" + 1
            SET VOTE_NUMBER TO VOTE_NUMBER + 1

            SET MAP TALLY "Option C" TO GET FROM MAP TALLY "Option C" + 1
            SET VOTE_NUMBER TO VOTE_NUMBER + 1

         END IF
      END IF
   END REPEAT

   DISPLAY "--- Final Results ---"
   SET OPTIONS TO GET ALL KEYS TALLY

   REPEAT FROM I = 1 TO SIZE OF OPTIONS
      SET OPTION TO OPTIONS[I]
      DISPLAY OPTION + ": " + GET FROM MAP TALLY OPTION + " votes"
